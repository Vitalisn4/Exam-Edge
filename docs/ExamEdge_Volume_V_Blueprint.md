## __2\.3 Key Observations from Paper 0765/2 for ExamEdge Design__

- __Mark allocation pattern:__ 6 questions at 3\-6 marks \(procedural/proof\); 4 questions at 7\-9 marks \(multi\-part, mixed methods\)\. Total: approximately 87 marks across 10 questions in 3 hours\. ExamEdge's exam simulation mode targets this ratio exactly\.
- __Numerical methods are explicitly tested:__ Newton\-Raphson \(Q5ii\) requires ExamEdge to generate questions where the student must: show a sign change, compute f'\(x\) symbolically, and apply the iteration formula with a given starting point\. The AI marking chain must verify each of these three steps independently with M1, M1, A1 marks\.
- __Proof\-type questions dominate:__ Mathematical induction \(Q9ii\), trig identity proof \(Q3i\), implicit differentiation proof \(Q8i\), and equivalence relation proof \(Q6ii\) together account for 17 marks\. ExamEdge's Socratic guidance engine is most important for these — a student who writes an incorrect base case or a vague transitive proof needs guided correction, not the answer\.
- __Log\-linear graph \(Q2\) requires student\-drawn graphs:__ This is a diagram\-handling requirement\. ExamEdge must provide a grid\-based graph plotting interface for this question type, or an alternative mechanism \(see Section 5\)\.
- __3D vectors \(Q7ii\) require spatial reasoning:__ Vector line equations, plane equations, and the angle between a line and a plane are high\-difficulty questions\. ExamEdge generates variants by changing the position vectors while ensuring the plane remains non\-degenerate\.
- __Periodic functions \(Q6i\) require sketch assessment:__ Graph sketching on a specified domain is a recurring requirement across A\-Level mathematics\. ExamEdge must handle sketch submission and evaluation\.

# __4\. Intelligent Assessment Architecture — Handwriting, Diagrams, Graphs, and Equations__

*The uploaded paper contains questions that require: graph plotting \(Q2\), geometric sketching \(Q6i, Q10b\), vector diagrams \(Q7ii\), and step\-by\-step algebraic working \(every question\)\. None of these can be adequately assessed by a text input box alone\. This section defines the complete multimodal assessment architecture\.*

## __4\.1 Assessment Input Modalities__

ExamEdge supports four answer submission modalities, each appropriate for different question types\. Students select the modality at the start of each question:

__Modality__

__Best For__

__When to Use__

__Technical Implementation__

__Typed text \+ KaTeX__

Algebraic manipulation, calculus, proof writing, short answers

KaTeX renders LaTeX notation client\-side\. MathQuill provides a WYSIWYG equation editor — students click to build equations without learning LaTeX syntax\. No server cost\.

__Digital handwriting \(stylus or touch\)__

Any question requiring freehand working — most mathematics and sciences

__Photo upload \(handwritten workbook\)__

Full worked solutions in a physical exercise book; scan/photo after completion

## __4\.2 Diagram and Graph Handling — Detailed Architecture__

### __4\.2\.1 Log\-linear graph questions \(Paper 0765/2, Q2\)__

Question 2 asks the student to draw a graph of log y vs log x, determine gradient and y\-intercept, and use these to find constants a and n for y = axⁿ\. This is a GCE Board staple that appears across Mathematics and Sciences\.

ExamEdge handles this via the Desmos graphing component embedded in the answer space\. The student:

1. Calculates log x and log y values from the given table \(typed into a provided data table input\)\.
2. The platform automatically plots the points on a Desmos\-rendered coordinate grid\.
3. The student draws the best\-fit line by clicking two points\.
4. ExamEdge extracts the gradient and y\-intercept from the student's line\.
5. The AI marking chain compares: gradient against expected gradient \(n\), y\-intercept against expected value \(log a\), and the final values of a and n against acceptable range\.

Mark scheme application: B1 for correct log transformation stated; M1 for correctly plotting; A1 for gradient within accepted tolerance \(±0\.1\); A1 for a and n to 1 d\.p\. as required by the board\.

### __4\.2\.2 Curve sketching questions \(Q6i, Q10b\)__

Curve sketching requires the student to demonstrate understanding of: intercepts, asymptotes, turning points, and general shape\. ExamEdge assesses sketches through a property\-based evaluation rather than pixel\-comparison:

- The AI is given the expected properties: x\-intercept at \(4,0\), y\-intercept at \(0,4\), vertical asymptote x=1, horizontal asymptote y=1 for y=\(x\-4\)/\(x\-1\)\.
- The student's sketch is evaluated against these properties using a combination of the interactive graph tool \(which can extract key coordinates\) and AI vision analysis of uploaded sketches\.
- Marks are awarded per property correctly demonstrated, not for sketch aesthetic quality\.

__Important:__ No sketch tool can perfectly replicate the cognitive experience of drawing on paper\. ExamEdge is transparent about this with students: 'For examination preparation, practising on paper remains important\. This tool helps you check that your sketch has the correct key features\.'

### __4\.2\.3 Vector diagrams and geometric constructions__

3D vector questions \(Q7ii\) involve spatial reasoning that benefits from visualisation\. ExamEdge provides a 3D vector visualisation component built with Three\.js that renders the position vectors A, B, and C in a rotatable 3D space\. This is not an assessment input — it is a learning aid\. The student completes the vector algebra in the equation editor; the 3D visualisation helps build spatial intuition\.

### __4\.2\.4 Biological and scientific diagrams__

Biology, Chemistry, and Physics questions often require diagram labelling \(cell diagrams, circuit diagrams, ray diagrams\)\. ExamEdge handles these through:

- Pre\-drawn diagram images with draggable label boxes that the student positions onto the correct structures\.
- For assessment: the label positions are compared to a predefined correct\-position schema\. Marks are awarded per correctly placed label\.
- For free\-hand diagram drawing \(e\.g\. drawing a circuit diagram from scratch\): photo upload path is the primary mechanism at MVP stage\. AI vision evaluates the key features present\.

## __4\.3 Handwritten Answer Submission — Two Pathways__

### __4\.3\.1 Pathway A: Post\-practice photo upload__

This pathway mirrors the real examination experience most closely\. The student:

1. Receives the question on\-screen in ExamEdge's exam simulation mode\.
2. Solves it in a physical exercise book using pen or pencil\.
3. When complete, photographs the page\(s\) with their phone camera\.
4. Uploads the photo\(s\) to ExamEdge via the mobile app camera interface\.
5. The photo is stored in Cloudflare R2 and processed by the AI vision pipeline\.
6. Claude claude\-sonnet\-4\-6 with vision analyses the handwritten working step\-by\-step against the mark scheme\.
7. Marks are awarded per step; specific feedback identifies where working was correct, where it was incomplete, and where errors occurred\.

Technical requirements: minimum photo resolution 1MP \(achievable by any Android phone released since 2014\)\. The AI vision pipeline is robust to moderate image rotation \(±15°\) and moderate lighting variation through pre\-processing \(contrast normalisation, perspective correction using OpenCV running in a Python microservice\)\.

### __4\.3\.2 Pathway B: Digital handwriting in\-app__

For the mobile app, ExamEdge provides a digital answer canvas using the React Native Canvas API \(or expo\-sketch library\)\. Students write directly on the phone screen with a finger or stylus\. The canvas captures strokes in real\-time\. When the student submits:

1. The canvas is exported as a PNG image\.
2. Google ML Kit's digital ink recognition \(free, on\-device\) performs OCR optimised for handwritten mathematical content\.
3. The transcribed text is passed through the KaTeX parser to extract mathematical structure\.
4. The structured answer is marked by the AI marking chain using the same mark scheme as typed answers\.
5. If ML Kit OCR confidence is below 0\.7, the original canvas image is sent to the AI vision pipeline as a fallback\.

__Honest limitation:__ Mathematical handwriting OCR, even with Google ML Kit and GPT\-4V\-class vision models, is not perfect on all handwriting styles\. ExamEdge communicates this clearly: 'If you believe your working was correctly recognised, tap to view the transcription\. If it was misread, upload a clearer photo or use the text input for this question\.'

## __4\.4 Digital Equation Editor — MathQuill \+ KaTeX__

ExamEdge uses MathQuill \(an open\-source WYSIWYG mathematical expression editor, MIT licence\) as the primary equation input interface\. MathQuill converts student interactions \(clicking fraction templates, square root symbols, integral signs\) into LaTeX notation, which KaTeX renders immediately\. Students who know LaTeX can type it directly; students who do not can use the visual toolbar\.

The MathQuill toolbar provides — relevant to the sample paper \(0765/2\):

- Fractions \(for partial fractions Q8ii, complex numbers Q5i\)
- Superscripts and subscripts \(for polynomials Q1, binomial Q4ii\)
- Summation notation \(for mathematical induction Q9ii, series Q4i\)
- Integral symbols \(for integration Q8ii, differential equations Q10a\)
- Trigonometric functions \(for trig identities Q3\)
- Logarithm notation \(for log\-linear graph Q2\)
- Vector notation with hats and bold \(for vectors Q7ii\)
- Matrix input grid \(for matrix operations Q9i\)
- Piecewise function notation \(for periodic function Q6i\)

# __5\. AI\-Powered Personalised Learning Ecosystem__

*ExamEdge is not an examination practice tool that also does tutoring\. It is a complete AI learning ecosystem that happens to culminate in examination readiness\. The distinction matters: the platform must be useful to a student on Day 1 of Year 1 of A\-Level, not just in the revision weeks before the examination\.*

## __5\.1 The Learning Journey Architecture__

ExamEdge maps to four distinct phases of a student's secondary school journey, each with a different platform function:

__Phase__

__Timing__

__Platform Function__

__Primary Student__

__Foundation Learning__

AI Tutor mode: teach concepts from first principles\. Topic\-by\-topic curriculum walkthrough\. Worked examples with step\-by\-step explanations\. Formative mini\-quizzes after each topic\.

Student who has just started the subject and needs guided instruction, not examination practice\.

__Concept Consolidation__

Targeted practice mode: questions at growing difficulty per topic\. Socratic guidance on errors\. Spaced repetition of earlier topics\. Cognitive Fingerprint building\.

__Examination Preparation__

3\-6 months before examination

Exam simulation mode \+ marking\. Past\-paper style questions at board standard\. Timed practice\. Weakness identification and targeted remediation\. Mock examination papers\.

Student who needs to convert understanding into examination performance under time pressure\.

__Final Revision__

Last 4\-6 weeks before examination

Intensive revision mode: high\-frequency spaced repetition of weak topics only\. Daily timed mini\-mocks\. Predicted grade tracking\. Daily examination countdown with targeted study plan\.

Student who needs to maximise mark gain in the time remaining\. No new content — only consolidation and retrieval\.

## __5\.2 AI Tutor: Teaching From First Principles__

The AI Tutor mode is the mechanism through which ExamEdge serves students in rural and underserved communities where qualified teachers may be absent, overloaded, or covering subjects outside their specialism\. The AI Tutor:

- __Presents concepts in progressive layers:__ A\-Level Pure Mathematics With Mechanics begins with the prerequisite algebraic fluency check before advancing to calculus\. The AI assesses what the student already knows before teaching what they do not\.
- __Uses culturally grounded examples:__ Mechanics problems use contexts from Cameroon: a car on a road in Yaounde, a ball thrown on a football pitch, a worker pushing a cart in a market\. This is not cosmetic — contextual familiarity reduces cognitive load and helps students map abstract concepts to physical intuition\.
- __Adapts explanation depth:__ If a student's first attempt at a concept shows foundational gaps \(e\.g\. inability to differentiate x² correctly before beginning integration by parts\), the AI steps back to the gap topic before proceeding\. No student is pushed forward while carrying unresolved misconceptions\.
- __Offers multiple explanation formats:__ Each concept explanation is available in three formats: concise \(key formula \+ one worked example\), standard \(derivation \+ two worked examples \+ common mistakes\), and extended \(full conceptual motivation \+ three worked examples \+ historical context\)\. Students choose the format; the AI recommends based on Cognitive Fingerprint\.

## __5\.3 Learning Gap Detection and Personalised Study Plans__

ExamEdge generates a personalised study plan for every student weekly\. The plan is driven by three data sources: the spaced repetition schedule \(which topics are due for review based on forgetting curve analysis\), the mastery map \(which topics have MVS below 0\.7\), and the examination countdown \(how many days remain and which topics are highest\-weight on the examination paper\)\.

For a GCE A\-Level Pure Mathematics With Mechanics student, the study plan might read:

*'This week's priority: 1\) Complex numbers — your mastery validation score is 0\.52\. Spend 40 minutes on modulus\-argument form and De Moivre's theorem\. 2\) Binomial theorem — due for spaced repetition \(last practised 18 days ago\)\. 15 minutes on term\-independent\-of\-x problems\. 3\) Trigonometric identities — you have not yet attempted R\-form questions\. 25 minutes with worked examples before practice\. Your GCE A\-Level is in 31 days\. At current pace: predicted grade B\. Completing this plan moves you to grade A range\.'*

## __5\.4 Special Considerations for Rural and Underserved Students__

The student in the north\-west region of Cameroon with intermittent electricity and mobile data priced at XAF 500 per 100MB deserves exactly the same quality of AI tutoring as a student in Yaounde with fibre internet\. ExamEdge's architecture for this student:

- PWA offline caching downloads the next 5 days of study sessions when connected\. The student studies offline for 5 days and syncs when they next access WiFi\.
- The AI tutor's concept explanations are generated once and cached — a student who downloads the Newton\-Raphson concept explanation tonight can re\-read it offline 15 times without any additional data cost\.
- The photo upload pathway for handwritten answers works offline — photos are queued in IndexedDB and uploaded when connectivity is available\.
- Text\-to\-speech \(Web Speech API, free, built into all modern Android browsers\) reads question text aloud for students with reading difficulties or who are studying while travelling\.
- The USSD companion interface \(Africa's Talking\) delivers daily revision prompts to students with feature phones for XAF 15 per day — less than the cost of a sachet of water\.

# __6\. Verifying Genuine Understanding — Mastery vs Memorisation__

*A student who has memorised the Newton\-Raphson formula and can apply it to the question in Paper 0765/2 Q5ii has not necessarily understood numerical methods\. They may fail if the next paper uses a slightly different form\. ExamEdge is designed to detect and close this gap\.*

## __6\.1 The Five\-Layer Understanding Test \(Applied to Paper 0765/2\)__

Using Question 5\(ii\) as a worked example — Newton\-Raphson applied to f\(x\) = x³ − 2x − 11:

__Level__

__What It Tests__

__ExamEdge Implementation__

__L1__

__Answer Correctness__

Can the student produce the correct second approximation \(x ≈ 2\.7\)?

30% of mastery credit\. Many students who copied a solution can reach this level\.

__L2__

__Method Transparency__

Can the student show: \(1\) sign change between f\(2\) and f\(3\), \(2\) f'\(x\) = 3x²−2, \(3\) Newton\-Raphson formula applied correctly?

Additional 25% credit\. Step\-by\-step M1/A1 marking\. A student who copied only the final answer gets zero M1 marks\.

__L3__

__Conceptual Explanation__

Can the student explain: why we need a sign change to show a root exists; what the Newton\-Raphson method geometrically represents \(tangent line approximation\)?

Additional 25% credit\. UVE probe: 'Explain in your own words why showing f\(2\) and f\(3\) have opposite signs is necessary before applying Newton\-Raphson\.'

__L4__

__Variant Transfer__

Can the student apply the same method to g\(x\) = x³ − 5x \+ 1, starting from x₀ = 2?

Additional 20% credit\. Parameterised variant generated automatically with different coefficients\. Requires same three steps; different numerical result\.

__L5__

__Error Identification__

'A student found f'\(x\) = 3x² − 2 but then applied the formula as x₁ = x₀ − f\(x₀\)/f\(x₀\)\. What is the error and what is the correct result?'

Bonus probe for students who complete all four layers\. Tests depth of understanding beyond standard mastery\.

## __6\.2 Proof Question Handling \(Mathematical Induction — Q9ii\)__

Proof questions are the highest cognitive demand question type in A\-Level mathematics\. They are also the question type most susceptible to superficial memorisation — a student who has seen one induction proof can often reproduce the structure without understanding it\.

ExamEdge's approach to proof question assessment for the result Σr\(3r\-1\) = n²\(n\+1\):

1. __Step 1 — Base case:__ The AI marks the base case \(n=1: LHS = 1×\(3×1\-1\) = 2; RHS = 1²×2 = 2; LHS = RHS\) as B1 if both sides are evaluated numerically and declared equal\.
2. __Step 2 — Inductive hypothesis:__ The AI marks the assumption 'Assume P\(k\) is true: Σr\(3r\-1\) = k²\(k\+1\)' as M1 if it is stated explicitly with the subscript variable changed from n to k\.
3. __Step 3 — P\(k\+1\) attempt:__ The AI marks M1 for a genuine attempt to write Σr\(3r\-1\) for r=1 to k\+1 as the k\-sum plus the \(k\+1\)th term\.
4. __Step 4 — Algebraic manipulation:__ The AI marks A1 if the student correctly simplifies k²\(k\+1\) \+ \(k\+1\)\(3\(k\+1\)\-1\) to \(k\+1\)²\(k\+2\)\. This is the hardest step and the most common error point\.
5. __Step 5 — Conclusion:__ A1 for a correctly stated conclusion: 'Therefore P\(k\+1\) is true\. Since P\(1\) is true and P\(k\) implies P\(k\+1\), by mathematical induction P\(n\) is true for all n ∈ ℕ\.'

Socratic guidance for a student who fails Step 4: 'You have correctly written the sum as k²\(k\+1\) \+ \(k\+1\)\(3k\+2\)\. What factor is common to both terms? Try factoring it out first\.' The AI never completes the factorisation — it identifies the key next step and asks the student to perform it\.
