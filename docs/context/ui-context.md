# UI Context — ExamEdge

UI documentation is split into three files for clarity (matching the starter kit pattern):

| File | Purpose |
|------|---------|
| `ui-tokens.md` | Colors, typography, spacing, component token values |
| `ui-rules.md` | Layout patterns, page structure, do-nots |
| `ui-registry.md` | Living component registry — update after every component built |

Read all three before any UI work. ui-registry.md prevents pattern drift across sessions.

**Quick reference:**
- Primary brand: Exam Blue `#1E40AF`
- Mobile-first: 360px, min 16px body, 44px touch targets
- Exam mode: paper white `#FFFEF7`, no navbar, fullscreen
- Mastery: red/amber/green per theta thresholds
- Mark badges: M1 blue, A1 green, B1 purple, denied red
